# Knowledge Storage

Knowledge uses vector stores to index and retrieve documents. Choose the right backend for your use case.

## Quick Setup

```python
from praisonaiagents import Agent

# Default: Chroma (zero-config, local)
agent = Agent(knowledge=["docs/"])

# With specific provider
agent = Agent(
    knowledge={
        "sources": ["docs/"],
        "vector_store": {
            "provider": "qdrant",
            "config": {"url": "http://localhost:6333"}
        }
    }
)
```

## Choosing a Vector Store

| Use Case | Recommended | Why |
|----------|-------------|-----|
| Local development | Chroma | Zero config, embedded |
| Production (hosted) | Pinecone | Managed, scalable |
| Production (self-hosted) | Qdrant/Weaviate | Performant, open-source |
| PostgreSQL stack | PGVector | Same infrastructure |
| Enterprise | Milvus | High availability |

## Configuration Examples

<Tabs>
  <Tab title="Chroma">
```python
agent = Agent(
    knowledge={
        "sources": ["docs/"],
        "vector_store": {
            "provider": "chroma",
            "config": {
                "collection_name": "my_docs",
                "path": ".praison/chroma"
            }
        }
    }
)
```
  </Tab>
  <Tab title="Qdrant">
```python
agent = Agent(
    knowledge={
        "sources": ["docs/"],
        "vector_store": {
            "provider": "qdrant",
            "config": {
                "url": "http://localhost:6333",
                "collection_name": "my_docs"
            }
        }
    }
)
```
  </Tab>
  <Tab title="Pinecone">
```python
agent = Agent(
    knowledge={
        "sources": ["docs/"],
        "vector_store": {
            "provider": "pinecone",
            "config": {
                "api_key": "your-api-key",
                "index_name": "my-index"
            }
        }
    }
)
```

## Quick Start

<Steps>
<Step title="SDK only â€” embed agents in your Python app">
```bash
pip install praisonaiagents
```
The core `Agent`, tools, memory, and hooks. No CLI, no gateway.
</Step>

<Step title="Terminal CLI â€” run/chat/code, no gateway or bots">
```bash
pip install praisonai-code
```
The agentic CLI (`run`, `chat`, `code`) and LLM runtime, without channel bots.
</Step>

<Step title="Gateway + bots â€” channel runtime without the wrapper">
```bash
pip install "praisonai-bot[gateway,bot]"
```
The WebSocket gateway, Telegram/Discord/Slack bots, and the scheduler tick.
</Step>

<Step title="Training â€” fine-tune LLMs or train agents">
```bash
pip install "praisonai-train[llm]"
```
LLM fine-tuning via Unsloth and iterative agent training, without the wrapper.
</Step>

<Step title="Browser â€” automate Chrome and the web">
```bash
pip install "praisonai-browser[all]"
```
The bridge server, CDP agent, and Playwright engine â€” usable on their own or inside the full `praisonai` install.
</Step>

<Step title="Everything â€” the full wrapper">
```bash
pip install praisonai
```
Transitively pulls `praisonai-code`, `praisonai-bot`, `praisonai-train`, `praisonai-browser`, and `praisonaiagents`.
</Step>
</Steps>

## How Tiers Interact at Runtime

The bot tier reaches wrapper-only features through a lazy bridge, so it stays installable on its own.

```mermaid
sequenceDiagram
    participant User
    participant Bot as praisonai-bot
    participant SDK as praisonaiagents
    participant Bridge as Lazy config bridge

    User->>Bot: message / scheduled tick
    Bot->>SDK: Agent.start(request)
    SDK-->>Bot: agent response
    Bot->>Bridge: resolve config (only if wrapper installed)
    Bridge-->>Bot: config or standalone default
    Bot-->>User: reply
```

The bot tier never imports the wrapper directly â€” the bridge resolves config the same way whether you run bot-tier standalone or the full `praisonai` install.

## Ownership Table

Each tier owns a clear slice and must not depend upward.

| Tier | Package | Owns | Must not depend on |
|------|---------|------|-------------------|
| 1 | `praisonaiagents` | Agent, tools, memory, hooks, protocols | any wrapper |
| 2a | `praisonai-code` | `run`/`chat`/`code`, Typer CLI, runtime, LLM | `praisonai` PyPI (lazy bridge only) |
| 2b | `praisonai-bot` | Bots, gateway, channel CLI, OS daemon, **gateway scheduler tick** | `praisonai` PyPI (lazy bridge for jobs/UI) |
| 2c | `praisonai-train` | LLM fine-tuning (Unsloth), iterative agent training, `train` CLI group | `praisonai` PyPI |
| 2d | `praisonai-browser` | Extension bridge, CDP agent, Playwright engine, browser CLI | `praisonai` PyPI |
| 3 | `praisonai` | Framework adapters (CrewAI/AutoGen), serve, dashboard, async jobs API | â€” |

**Publish order:** `praisonaiagents` â†’ `praisonai-code` + `praisonai-bot` + `praisonai-train` + `praisonai-browser` â†’ `praisonai`

## Config Kernel

Shared configuration lives in `praisonai_code/cli/configuration/`. The `praisonai-bot` tier reaches it through a lazy internal bridge, so the bot package stays installable on its own without importing the wrapper. You never call this bridge directly â€” it resolves config the same way whether you run bot-tier standalone or the full wrapper.

## Common Patterns

### Embed agents in my app

Install the SDK and call `Agent` directly.

```python
from praisonaiagents import Agent

agent = Agent(name="assistant", instructions="You are a helpful assistant.")
agent.start("Summarise the latest release notes.")
```

### Deploy a bot with scheduled updates, no CrewAI

Install the bot tier and run a scheduled agent tick â€” no wrapper required.

```python
import asyncio
from praisonaiagents import Agent
from praisonaiagents.scheduler import ScheduleRunner, FileScheduleStore
from praisonai_bot.scheduler import ScheduledAgentExecutor

agent = Agent(name="watcher", instructions="Summarise new alerts.")
runner = ScheduleRunner(FileScheduleStore("./schedules"))
executor = ScheduledAgentExecutor(runner=runner, agent_resolver=lambda _id: agent)

asyncio.run(executor.run_loop(interval=15.0))
```

## Best Practices

<AccordionGroup>
<Accordion title="Start with praisonai unless you know you want a lighter install">
The full wrapper pulls every tier and is the recommended default. Drop to `praisonai-code`, `praisonai-bot`, `praisonai-train`, `praisonai-browser`, or `praisonaiagents` only when you have a specific reason to trim dependencies.
</Accordion>

<Accordion title="Shims keep old imports working â€” you don't need to migrate immediately">
After the C9 module moves, the old `praisonai.scheduler.executor` and `praisonai.scheduler.condition_gate` paths remain importable as backward-compatibility shims when the wrapper is installed. New code should import from `praisonai_bot.scheduler.*`, but existing code keeps running.
</Accordion>

<Accordion title="The scheduler tick works standalone from bot tier; RunPolicy needs the wrapper">
`ScheduledAgentExecutor` ships in `praisonai-bot`, so scheduled ticks run without the wrapper. The `RunPolicy` safety gate stays in `praisonai` â€” install the wrapper if you need it for unattended runs.
</Accordion>
</AccordionGroup>

## Overview

PraisonAI includes a privacy-focused telemetry system designed to help improve the framework while respecting user privacy. Our telemetry implementation follows these core principles:

- **Privacy-first**: No personal data or user content is ever collected
- **Opt-out by default**: You can easily disable telemetry at any time
- **Transparent**: All telemetry code is open source and auditable
- **Minimal**: Only anonymous usage metrics are collected

<Note>
Telemetry helps us understand how PraisonAI is used in the real world, allowing us to prioritize features and improvements that matter most to our users.
</Note>

## Quick Start

Telemetry is enabled by default but can be easily disabled using environment variables:

<CodeGroup>
```bash Environment Variable
export PRAISONAI_TELEMETRY_DISABLED=true
```

```python Python Code

```

```yaml Docker Compose
services:
  praisonai:
    environment:
      - PRAISONAI_TELEMETRY_DISABLED=true
```
</CodeGroup>

## Configuration Options

### Environment Variables

PraisonAI respects multiple environment variables for controlling telemetry:

<Steps>
<Step title="PRAISONAI_TELEMETRY_DISABLED">
Set to `true` to disable telemetry completely. This is the primary method for opting out.
</Step>

<Step title="PRAISONAI_DISABLE_TELEMETRY">
Alternative environment variable for disabling telemetry. Set to `true` to opt out.
</Step>

<Step title="DO_NOT_TRACK">
Universal standard for privacy preferences. Set to `1` or `true` to disable telemetry.
</Step>
</Steps>

### Programmatic Configuration

You can also control telemetry programmatically:

```python
from praisonaiagents.telemetry import telemetry

# Check if telemetry is enabled
if telemetry.enabled:
    print("Telemetry is active")

# Disable telemetry at runtime
telemetry.disable()

# Enable telemetry (if previously disabled)
telemetry.enable()
```

## What Data is Collected

Our telemetry system collects only anonymous usage metrics:

<Accordion title="Anonymous Metrics">
- **Agent Executions**: Count of agent runs (no task details)
- **Task Completions**: Success/failure rates (no task content)
- **Tool Usage**: Which tools are used (no input/output data)
- **Error Tracking**: Error types and frequencies (no sensitive data)
- **Feature Usage**: Which features are adopted (anonymous counts)
- **Performance Metrics**: Execution times and resource usage
</Accordion>

<Warning>
We **never** collect:
- Personal information or user identifiers
- Task descriptions or content
- API keys or credentials
- File contents or paths
- Network locations or IP addresses
- Any data processed by your agents
</Warning>

### Custom Telemetry Backends

Implement custom telemetry collectors:

```python
from praisonaiagents.telemetry import TelemetryCollector

class CustomTelemetry(TelemetryCollector):
    def track_event(self, event_name: str, properties: dict = None):
        # Send to your custom backend
        pass
    
    def track_agent_execution(self, agent_name: str, duration: float):
        # Custom agent tracking
        pass

# Register your custom collector
from praisonaiagents.telemetry import telemetry
telemetry.set_collector(CustomTelemetry())
```

## Performance Monitoring

### Latency Tracking

Monitor agent and task execution times:

```python
from praisonaiagents.telemetry import telemetry

# Automatic latency tracking for agents
agent = Agent(
    name="processor",
    telemetry_enabled=True  # Enabled by default
)

# Manual timing for custom operations
with telemetry.timer("custom_operation"):
    # Your code here
    pass
```

### Resource Usage

Track memory and CPU usage:

```python
# Telemetry automatically tracks resource usage
# Access metrics programmatically
metrics = telemetry.get_metrics()
print(f"Memory usage: {metrics.memory_mb}MB")
print(f"CPU usage: {metrics.cpu_percent}%")
```

## Advanced Usage

### Custom Events

Track custom events in your applications:

```python
from praisonaiagents.telemetry import telemetry

# Track custom events
telemetry.track_event(
    "custom_workflow_completed",
    properties={
        "workflow_type": "data_pipeline",
        "success": True,
        "duration_seconds": 45.2
    }
)
```

### Telemetry Filters

Filter sensitive information:

```python
from praisonaiagents.telemetry import telemetry

# Add filters to exclude sensitive data
telemetry.add_filter(lambda event: 
    event["name"] not in ["sensitive_operation"]
)

# Add property filters
telemetry.add_property_filter(lambda key, value:
    key not in ["api_key", "password", "secret"]
)
```

### Export and Analysis

Export telemetry data for analysis:

```python
# Get telemetry metrics
metrics = telemetry.export_metrics()

# Export to different formats
telemetry.export_to_json("telemetry_data.json")
telemetry.export_to_csv("telemetry_metrics.csv")

# Get summary statistics
summary = telemetry.get_summary()
print(f"Total events: {summary['total_events']}")
print(f"Unique sessions: {summary['unique_sessions']}")
```

## Troubleshooting

### Common Issues

<Accordion title="Telemetry not disabling">
Ensure environment variables are set before importing PraisonAI:
```python
import os
os.environ["PRAISONAI_TELEMETRY_DISABLED"] = "true"

# Then import PraisonAI
from praisonaiagents import Agent
```
</Accordion>

<Accordion title="Performance impact">
Telemetry is designed to be lightweight. If you notice performance issues:
```python
# Reduce telemetry frequency
telemetry.set_sampling_rate(0.1)  # Sample 10% of events

# Or disable specific collectors
telemetry.disable_performance_tracking()
```
</Accordion>

<Accordion title="Debug telemetry">
Enable debug logging to see telemetry activity:
```python
import logging
logging.getLogger("praisonai.telemetry").setLevel(logging.DEBUG)
```
</Accordion>

## API Reference

### TelemetryCollector

Base class for telemetry implementations:

```python
class TelemetryCollector:
    def track_event(self, event_name: str, properties: dict = None) -> None
    def track_agent_execution(self, agent_name: str, duration: float) -> None
    def track_task_completion(self, task_name: str, success: bool) -> None
    def track_tool_usage(self, tool_name: str, success: bool) -> None
    def track_error(self, error_type: str, error_message: str) -> None
    def get_metrics(self) -> dict
    def flush(self) -> None
```

### MinimalTelemetry

Privacy-focused telemetry implementation:

```python
class MinimalTelemetry(TelemetryCollector):
    enabled: bool
    session_id: str
    
    def enable(self) -> None
    def disable(self) -> None
    def set_sampling_rate(self, rate: float) -> None
    def add_filter(self, filter_func: Callable) -> None
    def export_metrics(self) -> dict
```

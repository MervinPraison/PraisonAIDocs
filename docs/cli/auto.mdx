The `--auto` flag automatically generates an agents.yaml configuration from a natural language task description.

## Quick Start

```bash
praisonai --auto "Create a research team to analyze market trends"
```

## Usage

```bash
praisonai --auto "<task description>" [options]
```

## Options

| Option | Description |
|--------|-------------|
| `--auto` | Enable auto mode with task description |
| `--merge` | Merge with existing agents.yaml instead of overwriting |
| `--framework` | Framework to use (crewai, autogen, praisonai) |

## Examples

### Basic Auto Generation
```bash
praisonai --auto "Build a content creation pipeline"
```

### With Framework Selection
```bash
praisonai --auto "Create a data analysis team" --framework praisonai
```

### Merge with Existing Agents
```bash
praisonai --auto "Add a quality reviewer" --merge
```

## How It Works

1. **Task Analysis**: The LLM analyzes your task description
2. **Agent Generation**: Creates appropriate agents with roles, goals, and tools
3. **YAML Output**: Generates `agents.yaml` configuration file
4. **Execution**: Optionally runs the generated agents

## Generated Output

The auto mode creates an `agents.yaml` file with:
- Agent definitions (name, role, goal, backstory)
- Task descriptions
- Tool assignments
- Process configuration (sequential, parallel, etc.)
